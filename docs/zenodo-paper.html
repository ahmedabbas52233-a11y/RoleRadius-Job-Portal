Abstract

This paper presents RoleRadius, an open-source full-stack job portal that applies Term Frequency–Inverse Document Frequency (TF-IDF) cosine similarity to intelligently match job seekers with relevant vacancies. Built on Django REST Framework and React 18, the system provides a complete recruitment workflow for both candidates and recruiters, including job posting with rich metadata, a multi-stage application pipeline with automated status notifications, CV text extraction, and an AI-powered matching engine. The platform achieves sub-second match scoring, supports 63 diverse job listings across 8 companies in its seed dataset, and is delivered with 61 automated tests, Docker containerisation, and a GitHub Actions CI/CD pipeline. RoleRadius demonstrates that natural language processing techniques traditionally applied in information retrieval can be effectively repurposed for recruitment technology, offering a transparent, explainable, and computationally efficient alternative to black-box recommendation systems.

Keywords: job portal TF-IDF cosine similarity Django React information retrieval recruitment technology natural language processing full-stack web application open source

1. Introduction

Digital recruitment platforms have become the primary interface between employers and job seekers. Platforms such as LinkedIn, Indeed, and Reed collectively handle millions of job listings and applications daily. However, the majority of widely-used portals rely on basic keyword matching or proprietary opaque algorithms that candidates cannot interrogate or understand. This opacity creates friction: qualified candidates are missed because their CV phrasing does not exactly match a job description's vocabulary, while recruiters are overwhelmed with irrelevant applications.

This paper describes RoleRadius, an open-source recruitment portal that addresses this problem by applying TF-IDF cosine similarity — a well-established information retrieval technique — to transparently quantify the compatibility between a candidate's profile and a job listing. Unlike neural embedding approaches, TF-IDF is computationally lightweight, interpretable, and requires no GPU infrastructure, making it suitable for a production educational deployment.

The contributions of this work are: (1) a complete, deployable full-stack recruitment platform; (2) an AI matching engine with documented time complexity; (3) a comprehensive test suite; and (4) a production-ready Docker deployment with CI/CD. All source code is released under the MIT licence.

2. Statement of Need

Existing open-source job boards (Jobboard.io, Django Jobs, Henning) provide basic CRUD interfaces without any intelligent matching. 3. System Architecture

RoleRadius follows a decoupled three-tier architecture: a React single-page application communicates with a Django REST API over HTTP, which persists data to a PostgreSQL database. File storage is handled by Cloudinary (with a local filesystem fallback for development). Figure 1 illustrates the full system topology.

Figure 1. Full system architecture of RoleRadius. The React frontend communicates with the Django REST API via a Vite development proxy. Tokens are stored in httpOnly cookies, eliminating the localStorage XSS vulnerability present in many tutorial implementations. The ML matching engine sits inside the Django process and is triggered on application submission.

4. Database Design

5. AI Matching Algorithm

5.1 TF-IDF Cosine Similarity

The matching engine in matching/engine.py applies TF-IDF vectorisation followed by cosine similarity to produce a compatibility score s ∈ [0, 100] for any candidate–job pair. The candidate text corpus is constructed from their headline, biography, skills list, and CV extracted text. The job corpus combines the title, description, requirements, skills required, and category.

Formally, given a candidate document c and a job document j, each tokenised into bigrams with stop words removed:

s(c, j) = 100 × cos(TF-IDF(c), TF-IDF(j)) = 100 × (TF-IDF(c) · TF-IDF(j)) / (‖TF-IDF(c)‖ × ‖TF-IDF(j)‖)

The vocabulary is constrained to a maximum of 8,000 features to bound memory usage. The use of bigrams (n-gram range 1–2) captures compound skills such as "machine learning" and "unit testing" that unigram-only approaches would split.

8. Testing

10. Application Status Pipeline

The application lifecycle is modelled as a directed state machine with seven states. Every transition is logged immutably in the ApplicationStatusHistory table, providing a full audit trail. Django signals fire candidate notification emails on every recruiter-driven transition.

Figure 5. Application status pipeline. Candidates progress through up to five stages; at any point the recruiter may mark the application as Rejected, or the candidate may Withdraw. Every transition creates an immutable ApplicationStatusHistory record and triggers an email notification to the candidate via Django's signals framework.

12. Known Limitations

The matching engine has two documented limitations relevant to production deployment. First, get_matched_jobs_for_candidate() loads all active jobs into Python memory (list(Job.objects.filter(is_active=True))), giving O(N) space complexity. This is acceptable at the scale of a university project but would require pre-computed score caching with Redis and Celery for deployments exceeding ~10,000 jobs. Second, TF-IDF does not model semantic similarity: "developer" and "engineer" are treated as completely distinct terms. A production system would benefit from word embedding similarity (Word2Vec, fastText) or sentence transformers to handle synonym resolution.

13. Conclusion

RoleRadius demonstrates that natural language processing techniques from information retrieval can be effectively applied to recruitment matching in an open, transparent, and computationally efficient manner. The system provides a complete, tested, and deployable recruitment portal that serves as both a practical tool and an educational reference for full-stack development with AI integration.
